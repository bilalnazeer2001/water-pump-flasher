# Phone / browser flasher

Flash the Water Pump firmware from a browser (Chrome/Edge on a PC, or
Android Chrome with a USB-OTG cable) — no laptop toolchain, no IDE.

## One-time per release: build the merged image

1. Build the firmware on your dev PC:
   ```
   pio run
   ```
2. Merge bootloader + partition table + app into one file:
   - Windows:  `flasher\merge.bat`
   - Mac/Linux: `bash flasher/merge.sh`

   This writes `flasher/merged.bin`.

3. Publish the `flasher/` folder (it already contains `index.html`,
   `manifest.json`, and now `merged.bin`). Easiest: push to a GitHub repo
   and enable **GitHub Pages**. You get a URL like
   `https://<you>.github.io/<repo>/flasher/`.

## Flashing a device

1. Open the published page in **Chrome/Edge**.
   - Android: connect the ESP32 with a **USB-OTG cable** first.
2. Tap **Connect & Flash**, choose the USB serial port.
3. Wait for "flashing complete".

This first flash **erases everything** and installs the dual-OTA layout.
After that, every update is wireless: hold the config button → join the
**WaterPump-Setup** Wi-Fi → open the setup page → upload the new
`firmware.bin` (NOT merged.bin) under "Firmware update (OTA)".

## Files

| File          | What it is                                            |
|---------------|-------------------------------------------------------|
| index.html    | The installer web page (ESP Web Tools)                |
| manifest.json | Tells the installer which chip + which image to flash |
| merged.bin    | Generated by merge.bat/merge.sh — the full image      |

## Notes

- iPhones cannot flash over USB — use Android or a PC for the first flash.
- For wireless OTA you upload `firmware.bin` (~1 MB), not the merged image.
- If `esptool.py` isn't on PATH, run it via PlatformIO:
  `pio pkg exec -- esptool.py ...` or `python -m esptool ...`.
